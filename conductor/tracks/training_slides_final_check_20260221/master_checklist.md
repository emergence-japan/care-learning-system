# 研修スライド品質保証チェックリスト (Master Checklist)

各科目のシードファイル（`prisma/seeds/*.ts`）の品質を確認するための2軸のチェックポイントです。

## 1. デザイン・構造軸（HTML/UI）
- [ ] **導入テンプレート:** `flex flex-col items-center justify-center text-center` の構成を含んでいるか。
- [ ] **バッジ演出:** `bg-blue-700.text-white.px-6.py-2.rounded-full` 等のバッジコンポーネントが適切に使われているか。
- [ ] **重要事項 (MUST CHECK):** `animate-ping` を含む警告演出が正しく記述されているか。
- [ ] **ケーススタディ:** `bg-amber-50` や `border-dashed` などの事例専用スタイルが適用されているか。
- [ ] **まとめ演出:** `rotate-3` のアイコンボックスを含む、清潔感のある終了画面になっているか。
- [ ] **角丸基準:** 主要な要素に `rounded-[2rem]` 以上（または `3xl`, `full`）が適用されているか。
- [ ] **視覚的階層:** `font-black` が重要な見出しに使用され、メリハリがついているか。
- [ ] **レスポンシブ対応:** `lg:` プレフィックスによるPC版レイアウトと、スマホ版の出し分け（`<br class="lg:hidden" />`）が考慮されているか。

## 2. ID・学習設計軸（Instructional Design）
- [ ] **冒頭のフック (ARCS - Attention):** 最初のスライドや導入文に、受講者をドキッとさせる問いかけ（「？」）があるか。
- [ ] **法的・社会的背景 (ARCS - Relevance):** 2024年度の法改正や、義務化の背景、実施しない際のリスクが明示されているか。
- [ ] **目標の提示:** 「何ができるようになるか」を具体的に示した学習目標セクションがあるか。
- [ ] **事例解説 (ガニェ - 学習の指針):** 解説だけでなく、現場での具体的な場面（ケーススタディ）を最低1つ含んでいるか。
- [ ] **アクションプラン (ガニェ - 移転の促進):** 研修の最後に、明日（今日）から現場でできる具体的なアクションが示されているか。
- [ ] **整合性 (Instructional Alignment):** 理解度テストの正解が、スライドの重要強調箇所と論理的に一致しているか（スライドに答えが書いてあるか）。
- [ ] **テスト品質:** 理解度テストが10問、各4つの選択肢と詳細な解説を備えているか。

## 3. 分量・形式
- [ ] **スライド枚数:** 約25枚前後（最低23枚）の構成になっているか。
- [ ] **ファイル形式:** `export async function seedXxx(prisma: PrismaClient)` の形式で独立しており、既存のシードと衝突しないか（slugの一意性）。
