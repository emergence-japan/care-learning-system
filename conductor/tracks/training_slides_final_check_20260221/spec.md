# トラック仕様書: 研修スライドの最終確認 (training_slides_final_check)

## 1. 概要
全15科目の研修コンテンツ（スライドおよびテスト）が、厚生労働省の定める法定研修の要件を満たし、かつ「高齢者虐待防止研修（01_abuse.ts）」と同等の極めて高いデザイン品質と学習効果（ID基準）を備えていることを最終保証する。不備がある科目は、マスターの構造を継承して再構築する。

## 2. 品質基準

### 2.1 デザイン・構造要件（Master Format: 01_abuse.ts 準拠）
「高齢者虐待防止研修（01_abuse.ts）」のHTML構造とUI/UX設計を「唯一の正解」とし、全科目に適用する。

- **コンポーネントの継承:**
  - **導入（Intro）:** 中央揃えのタイトル、太字メッセージ、薄いブルーの引用ボックス。
  - **学習目標（Objectives）:** 丸数字付きカードのグリッドレイアウト。
  - **重要事項（Must Check）:** `animate-ping` 付きの赤い警告ドットと強調カード。
  - **分類（Categories）:** パステルカラー（青・赤・琥珀・紫・緑）で色分けされたボタン風カード。
  - **ケーススタディ:** 琥珀色の点線枠（`border-dashed`）と「CASE STUDY」バッジ。
  - **まとめ:** 傾き（`rotate-3`）のついた大きなアイコンと清潔感のある終了画面。
- **視覚的ルール:**
  - **タイポグラフィ:** `font-black`（超極太）によるメリハリのある視覚階層。
  - **造形:** `rounded-[2rem]` 以上の大きな角丸、`blur-3xl`（ぼかし）による奥行き演出。
- **アクセシビリティ:**
  - PC（`lg:`）とスマホでのレイアウト切り替えロジックの完全模倣。
  - PC表示時の「単語単位の改行」による可読性確保。
- **枚数:** 1科目あたり約25枚を基準とする。

### 2.2 ID（学習設計）要件
受講者の「行動変容」を促すため、以下の学術的フレームワークを実装する。

- **ARCSモデル（動機付け）:**
  - **Attention（注意）:** 冒頭に現場の葛藤や「ドキッとする問いかけ」を配置し、自分事化させる。
  - **Relevance（関連性）:** なぜこの知識がプロとして不可欠か、リスク回避にどう繋がるかを明示する。
- **ガニェの9教授事象（構成）:**
  - **目標提示:** 冒頭で「何ができるようになるか（行動目標）」を具体的に示す。
  - **学習の指針:** ケーススタディ（事例解説）を必ず含め、理論を現場の振る舞いに変換する。
  - **移転の促進:** 最後に「明日から現場でできるアクションプラン」を提示する。
- **教育的整合性（Instructional Alignment）:**
  - **目標・内容・評価の一致:** スライドで強調した重要語句や基準が、そのまま理解度テストの正解根拠となっていること。

## 3. 手順
1. **01_abuse.ts の構造分析:** マスターのHTML構造とID要素を完全に把握する。
2. **科目別検証:** 各科目のシードファイル（`prisma/seeds/*.ts`）を読み込み、マスターとの乖離（デザインの簡素化、事例の不足、テストとの不一致）を特定する。
3. **修正・再構築:** 不備がある科目は、マスターのHTMLテンプレートをベースに内容を流し込み、ID要素（フック、事例、まとめ）を補強する。
4. **DB反映と実機確認:** シードを再実行し、管理画面プレイヤーでPC/スマホ両方の表示・動作を確認する。

## 4. 完了定義 (Acceptance Criteria)
- [ ] 全15科目が `01_abuse.ts` と同等のデザイン・アニメーション・レスポンス品質を備えている。
- [ ] すべての科目に「当事者意識を高めるフック」と「具体的なケーススタディ」が含まれている。
- [ ] `14_prevention.ts` が「介護予防」の内容としてマスター準拠で新規作成されている。
- [ ] テストの設問がスライド内容と完全に一致し、学習の振り返りとして機能している。
- [ ] PC/スマホ両方の環境でレイアウト崩れがなく、単語単位の改行が適切に行われている。

## 5. スコープ外
- スライドプレイヤー自体の新規機能開発（既存のバグ修正は含む）。
- 15科目以外のコンテンツ追加。
