# トラック仕様書: 研修スライドの最終確認 (training_slides_final_check)

## 1. 概要
全15科目の研修コンテンツ（スライドおよびテスト）が、厚生労働省の定める法定研修の要件、および独自に定義するID（インストラクショナルデザイン）基準を完全に満たしていることを最終確認し、発見された不備をすべて修正する。
また、重複していた「身体拘束」の1科目（14_prevention.ts）を「介護予防及び要介護進行予防」に差し替え、法定研修の網羅性を確保する。

## 2. 機能要件・確認基準

### 2.1 ID要件の定義（マスターフォーマット）
本トラックの開始時に以下の項目を「標準規格」として明文化し、全科目の検証基準とする。
- **デザイン:** 白基調の清潔感のあるデザイン（高齢者虐待防止研修をモデルとする）。
- **枚数:** 1科目あたり約25枚を基準とする。
- **構成:** 冒頭に「フック（問いかけ）」を配置し、学習者の関心を惹く構成。
- **アクセシビリティ:** PCおよびスマホの両方で、単語単位の改行やレイアウト崩れがないこと。

### 2.2 厚労省要件の網羅性
- 各科目の法的根拠に基づいた必須知識がすべて含まれているか（形式的な整合性よりも内容の網羅性を優先）。
- **特記事項:** `14_prevention.ts` を「介護予防及び要介護進行予防に関する研修」の内容に完全に作り変える。

### 2.3 理解度テストの品質
- スライドの内容を理解していれば正解できる設問か。
- 誤字脱字、選択肢の不備がないか。

### 2.4 表示・動作確認
- 管理画面のスライドプレイヤー上での実機表示確認（レスポンシブ対応）。
- 前後の遷移、テストの回答、修了判定の動作が正常か。

## 3. 手順
- 管理画面（スライドプレイヤー）上での目視確認を主とする。
- 1科目ずつチェックリストに基づき確認を行い、不備があればシードファイル（`prisma/seeds/*.ts`）を直接修正し、DBを更新する。

## 4. 完了定義 (Acceptance Criteria)
- [ ] 全15科目の各スライドが、定義されたID要件を満たしている。
- [ ] `14_prevention.ts` が「介護予防」の内容に更新されている。
- [ ] 厚労省要件の網羅性が確認され、不足があれば補完されている。
- [ ] テストの設問とスライド内容が一致し、正常に動作する。
- [ ] PC/スマホ両方の環境で表示崩れがないことが確認されている。
- [ ] 発見された誤字脱字、表現の不備がすべて修正され、DBに反映されている。

## 5. スコープ外
- スライドプレイヤー自体の新規機能開発（既存の表示バグ修正は含む）。
- 全く新しい科目の追加（「介護予防」の差し替えを除く）。
