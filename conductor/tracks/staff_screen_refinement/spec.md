# Track Specification: Staff Screen Refinement

受講者（スタッフ）が日常的に利用する学習画面（ダッシュボード、コース一覧、学習体験）を、モバイルファーストでモダンなUI/UXに刷新する。

## 目的
- スタッフの学習意欲を高め、義務感を軽減する。
- スマートフォンでの操作性を最優先し、現場の隙間時間でもストレスなく学習できる環境を提供する。
- 学習の進捗状況を一目で把握できるようにする。

## 対象ユーザー
- 介護現場のスタッフ（主にスマートフォンまたはタブレットで利用）

## 主要機能とデザイン要件

### 1. ダッシュボード (ホーム画面)
- **ヒーローエリア**:
  - 現在の受講状況（全体の完了率）を視覚的なリングチャートやバーで表示。
  - 「次に受講すべきコース」へのダイレクトアクセス（ワンタップで再開）。
  - モチベーションを高めるメッセージ（例: 「あと少しで今月の目標達成です！」）。
- **タスクリスト（研修計画）**:
  - カード型のデザインを採用。
  - 「期限切れ」「期限間近」などのアラートを分かりやすく、しかし威圧感なく表示。
  - ステータス（未着手・受講中・完了）によるフィルタリングやタブ切り替え（必要に応じて）。

### 2. コース詳細・受講画面
- **コース概要**:
  - イントロダクションを見やすく表示。
  - 所要時間の目安や、獲得できるスキル（バッジなど）を表示。
- **学習体験 (SlidePlayer連携)**:
  - 既存の `SlidePlayer` コンポーネントへのスムーズな遷移。
  - 全画面モードの活用。

### 3. モバイルナビゲーション
- 下部固定のナビゲーションバー（ボトムタブバー）の導入を検討（ホーム、コース一覧、設定/ログアウト）。
- 指の届きやすい位置にアクションボタンを配置。

## 技術スタック
- **Frontend**: Next.js (App Router), Tailwind CSS, Framer Motion (アニメーション)
- **Components**: Lucide React (Icons), Shadcn UI (Base components)

## 実装ステップ
1.  **モバイルナビゲーションコンポーネントの作成**: `MobileNav` (既存確認、なければ作成)。
2.  **ダッシュボードUIの刷新**: `src/app/page.tsx` のレイアウト変更。
3.  **コース詳細・一覧のUI調整**: 必要に応じて。

## デザインの方向性
- **Clean & Airy**: 余白を多めに取り、圧迫感のないデザイン。
- **Focus**: 今やるべきことに集中できる、ノイズの少ない画面。
- **Friendly**: 業務的な冷たさを排除し、温かみのある（しかしプロフェッショナルな）トーン。
